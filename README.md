# Haphazard Solutions — website

The source for [haphazardsolutions.com](https://haphazardsolutions.com), a
[Jekyll](https://jekyllrb.com/) site served by **GitHub Pages**. Pages builds it
with its pinned `github-pages` gem (Jekyll 3.10, kramdown, Rouge). The custom
domain is set via `CNAME`, and HTTPS is enforced.

## How it's structured

```
_config.yml              Site settings — title, nav, email, URL, collections
CNAME                    Custom domain (haphazardsolutions.com) — don't delete
Gemfile                  Gems for local preview only; Pages uses its own pinned env

index.html               Home
services.html            Services — Operational Intelligence + Operational Effectiveness
research/index.html      Research — index of Labs
research/neuraq.html     NeuraQ Lab
about.html               About
knowledge.html           Knowledge base — searchable library of papers + articles

_layouts/hs.html         The shell every page uses (status strip, nav, footer, fonts)
_layouts/article.html    Renders Markdown articles; chains hs.html

assets/css/haphazard.css   The one stylesheet — the "instrument-panel" design
                           system. Re-theme by editing the :root tokens at the top.
assets/js/mermaid.min.js   Self-hosted Mermaid; loads only on `mermaid: true` pages
assets/favicon.svg         Browser-tab icon (bars-in-ring mark)
assets/logo-mark.png       Logo shown in the nav and footer

_articles/*.md           Knowledge-base articles → /knowledge/<slug>/
_data/kb.json            Generated whitepaper index (title, abstract, full text)
kb/*.pdf                 Whitepaper PDFs — served for download at /kb/
scripts/build_kb.py      Regenerates _data/kb.json from the PDFs in kb/
```

## Editing content

- **Text & pages:** edit the `.html` files. The content lives below the `---`
  front-matter block at the top of each file.
- **Navigation:** edit the `nav:` list in `_config.yml`.
- **Colors / branding:** edit the token variables in `:root` at the top of
  `assets/css/haphazard.css` — change the tokens, not the rules.

Every push to the `main` branch rebuilds and republishes the site (usually live
within a minute).

## Knowledge base

`/knowledge/` is one searchable library merged from two sources; full-text search
and the document count are derived from both at build time.

- **Articles** — Markdown files in `_articles/`, rendered to `/knowledge/<slug>/`
  via the `article` layout. Front matter: `title`, `date`, `description` (add
  `mermaid: true` to enable diagrams on that page). Drop in a file and it joins
  the library automatically.
- **Whitepapers** — PDFs in `kb/`, downloadable at `/kb/`. Their searchable index
  (`_data/kb.json`) is generated by `scripts/build_kb.py`, which shells out to
  `pdftotext`. Re-run it and commit the updated JSON whenever the PDF set changes.

## Previewing locally (Docker — nothing installs on your machine)

The site runs in a container; gems live in a Docker volume (`hs-bundle`), so
nothing lands on the host. Matches the GitHub Pages build (Jekyll 3.10 /
github-pages gem).

First-time setup — install gems into the volume:
```sh
docker run --rm -v "D:/dev/hs:/site" -w /site -v hs-bundle:/usr/local/bundle \
  ruby:3.1 bundle install
```

Start the live server (auto-rebuilds on save):
```sh
docker run -d --name hs-jekyll -v "D:/dev/hs:/site" -w /site \
  -v hs-bundle:/usr/local/bundle -p 4000:4000 \
  ruby:3.1 bundle exec jekyll serve --host 0.0.0.0 --force_polling
```
→ http://localhost:4000

Everyday controls:
```sh
docker stop hs-jekyll     # pause
docker start hs-jekyll    # resume (gems stay cached in the volume)
docker restart hs-jekyll  # required after _config.yml or new-collection changes
docker logs -f hs-jekyll  # watch build output
docker rm -f hs-jekyll    # remove container; volume + source untouched
```

`jekyll serve` does **not** hot-reload `_config.yml` — restart the container
after editing it (or after adding a new file to a collection).

## Custom domain & email

- DNS is configured at the registrar; `CNAME` points the domain at Pages and
  HTTPS is enforced (plain http redirects to https).
- `contact@haphazardsolutions.com` is a catch-all that forwards to an external
  mailbox — no mail server to run.
